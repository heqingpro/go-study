# 系统行情介绍

### 一、概览

本文档主要介绍数字货币行情常见概念以及对接交易所行情模块各功能相关约定。主要目的为为开发人员快速熟悉行情相关基础概念，以及介绍对接行情时的接口返回规范和思路方法。

本文主要分为两个部分，其中第二章为货币市场行情相关的一些基础介绍如相关名词概念等，第三章为行情模块对接参考介绍如返回数据样例、可参考的已对接交易等。

### 二、市场行情介绍

#### 2.1、名词解释

- orderbook： orderbook为交易所某一数字货币交易对当前时刻市场委托交易的情况。分为买盘和卖盘，每一行显示了价格和数量等信息。

- 历史交易（trades）：为交易所每笔订单的成交信息。一般为交易所市场的最新成交模块，包含交易价格和数量等信息。

- K线（kline）：主要分时描述某一交易对的交易价格变动趋势。

- 交易类型（现货spot、期货future、期权option）：
  
  - 现货交易即常见的币币交易，以当前时间点的价格交易货币。
  
  - 期货交易也叫合约交易，该合约约定了在未来某一个特定的时间（交割日）以确定的价格（期货合约价格）交易一定数量（合约规模）的某种商品。分为永续合约和到期合约。
  
  - 期权交易：期权也叫选择权，是买方通过支付卖方一定期权费从而获得在一定时间内以一定的协议执行价格购买或出售的权利，而卖方则必须向买方履行出售或购买的义务。某天之后期权失效，这一天称为到期日或成熟日。
    
    交易所（derbit）

- 费率（rate）一般指交易成交所需的费率

- 持仓量  反映交易所市场总仓量，交易规模。

- 交易对（symbol 例：BTCUSDT）一般为 （交易货币-定价货币）模式。

- 常见参数：
  
  - amount in contracts 期货合约数量
  - amount in coins  期货合约总数折合的结算货币数量。算法需根据不同的交易所算法，一般的算法为合约数量乘合约相应比率再除以合约价格。
  - exchange 交易所
  - contract_type 交易类型:（现货spot、期货future、期权option）
  - data_type 数据类型
  - bids 买方报价
  - asks 卖方报价

### 三、行情业务对接说明

一般的交易所行情对接有两种类型：一种是常见的根据用户定价下单，通常行情需要实现orderbook、price、trades模块接口；一种是根据报价流来下单的，通常行情就要实现quotestream接口。

#### 3.1 orderbook

orderbook根据交易类型分为现币和合约交易两种，根据交易所类型返回数据类型分为快照数据和更新数据两种，不同种类的orderbook需要的处理方式都不一样。开发对接时需要根据实际情况来判断对接。

- 备注：测试orderbook 接口时需要打开相应交易所的市场界面，对照交易所orderbook数据，看二者一段时间内的数据一样时才行。

##### 3.1.1 快照（SNAPSHOT）

此类型的交易所接口会返回某一时刻的orderbook快照，包含完整的orderbook信息。系统会根据返回的数据进行去重操作。

参考交易所：jex

- spot
  
  - 输出样例
    
    ```javascript
    let    metadata = {
            "bids" : [[price , amount]], 
            "asks" : [[price , amount]], 
            "timestamp": YYYYMMDDHHmmssSSS
        }
    let orderbook_market_data_event =
                {
                    exchange: this.name,
                    symbol: symbol,
                    contract_type: contract_type,
                    data_type: MARKET_DATA.ORDERBOOK,
                    metadata: metadata,
                    update_type: ORDERBOOK_UPDATE_TYPE.SNAPSHOT,
                    subscribed_orderbook_depth: this.api_config.orderbook_max_depth,
                    timestamp: utils._util_get_human_readable_timestamp()
                };
                global.EventHandler.emit(EVENT.ON_MARKET_DATA_ORIGIN, orderbook_market_data_event);
    ```

- future
  
  - 输出样例
    
    ```javascript
    let    metadata = {
            "bids" : [[price , amount in contracts, amount in coins]], 
            "asks" : [[price , amount in contracts, amount in coins]], 
            "timestamp": YYYYMMDDHHmmssSSS
    ```

##### 3.1.2 更新（quote_stream）

此类型数据交易所接口一般第一次返回的数据为完整的快照数据，后续的数据为更行的数据，有可能是相同报价的数量改变或者是之前的报价数量变为0。需要更新信息。

参考交易所：okcoin

- spot
  
  - 输出样例
    
    ```javascript
    if(data_type === "insert"){
    
       let metadata = {
            "bids" : [[price , amount]], 
            "asks" : [[price , amount]], 
            "timestamp": YYYYMMDDHHmmssSSS
        }
        let orderbook_market_data_event = {

            symbol: info.symbol,

                //数量等于0之前该价位报价变为0，需要删掉.arr[2]置0

        }
        let quote_market_data_event = {
            exchange: this.name,
            symbol: info.symbol,
            contract_type: info["contract_type"],
            data_type: MARKET_DATA.QUOTE,
            metadata: metadata,
            update_type: ORDERBOOK_UPDATE_TYPE.QUOTE_STREAM,
    ```

#### 3.2 trades

反映的是交易所最近的成交记录，系统会根据emit的数据进行去重操作，对接时也需要根据特殊情况考虑是否需要取到数据时直接进行去重操作。

- spot
  
  - 输出样例
    
    ```javascript

        exchange: this.name,
        contract_type: info.contract_type,
        symbol: info["symbol"],
        data_type: MARKET_DATA.TRADE,
        metadata: metadata,
        timestamp: utils._util_get_human_readable_timestamp()
    };
    
    global.EventHandler.emit(EVENT.ON_MARKET_DATA_ORIGIN, price_market_data_event);
    ```

#### 3.3 latest_price 最新价格

获取交易对的最新价格。没有直接提供最新i价格接口的话也可以通过K线接口或者ticker接口获取数据。

- 输出样例
  
  ```javascript
  let price_market_data_event={
      exchange: this.exchange,
      symbol: symbol,
      contract_type: contract_type,

          {
              result: true,
              last_price: price,
              time: moment(timestamp).format("YYYYMMDDHHmmssSSS")
          },
      timestamp: utils._util_get_human_readable_timestamp()
  };
  global.EventHandler.emit(EVENT.ON_MARKET_DATA_ORIGIN, myevent);   
  ```

返回交易对的报价流，一般交易所需要根据这个报价下单。

  ```javascript
  let metadata;
  metadata.result = true;
  metadata.account_id = account_id;
  //买入价格
  metadata.bids = [];
  //卖出价格
  metadata.asks = [];
  if(direction === TRADE_SIDE.BUY){
      order.bids.push(parseFloat(res.result.price/quantity),quantity);
  }else{
      order.asks.push(parseFloat(res.result.price/quantity),quantity);
  }
  let event = {
      exchange: this.name,
  ```
